## 1. API 요청 흐름 (정상 경로)

```
Client
  │
  ├─ POST /v1/agent/boot ──────────────────────────► handle_boot()
  │         (무인증, DID 발급)                            │
  │                                          [✅ M5] 필드 길이 + github_login charset 검증
  │                                                    └─ AgentRecord 생성 + WELCOME_CREDITS
  │
  ├─ Authorization: Bearer did:helm:<pubkey>
  │         │
  │         ▼
  │   require_auth() middleware
  │         │── DID 레지스트리에 존재? → 401
  │         │── [✅ C1] ed25519 X-Helm-Signature 검증 (write ops)
  │         │── [✅ C6] Rate limit 체크 (100 req/60s 슬라이딩 윈도우)
  │         │── CallerDid 주입 → 핸들러 전달
  │         │
  │         ├─ /v1/sense/cortex ────────────────────► handle_cortex()
  │         │         [✅ C7] knowledge_context max 50×4096 chars
  │         │
  │         ├─ /v1/sense/memory/:key ───────────────► handle_memory_*()
  │         │         [✅ M1] MAX_KEYS=10K / [✅ M2] KEY_LEN≤256
  │         │         [✅ M3] serialization 실패 명시적 에러
  │         │
  │         ├─ /v1/synco/stream ─────────────────────► handle_synco()
  │         │         [✅ C8] 50MB 상한 적용
  │         │
  │         ├─ /v1/agent/:did/credit ───────────────► handle_fico()
  │         │         [✅ C5] 소유자만 전체 / 타인은 score+band만
  │         │         [✅ L3] FICO api_call cap=10K (farming 방지)
  │         │
  │         ├─ /v1/agent/:did/earnings ─────────────► handle_earnings()
  │         │         (소유자 체크 이미 존재 ✓)
  │         │
  │         ├─ /v1/pool + /v1/pool/:id/join ─────────► handle_*_pool()
  │         │         [✅ C3] 잔고 검증 + 차감
  │         │         [✅ C4] monthly_cost NaN/Inf/상한 차단
  │         │         [✅ M6] MAX_POOL_MEMBERS=1000
  │         │
  │         └─ /v1/marketplace/post + /apply ────────► handle_*_post()
  │                   [✅ H5] title≤200 / description≤4096
  │                   [✅ H6] proposal≤2048
  │                   [✅ M7] MAX_APPLICATIONS=100
  │                   [✅ L1] log injection 방지 (\r\n → space)
  │
  └─ GET /v1/leaderboard (공개) ──────────────────────► handle_leaderboard()
```

## 2. 인증 레이어 상세 (패치 후)

```
요청 도착
    │
    ▼
┌─────────────────────────────────────────────────────┐
│  require_auth() — auth.rs                           │
│                                                     │
│  1. Authorization 헤더 파싱                          │
│     "Bearer did:helm:<base58_pubkey>"               │
│                                                     │
│  2. DID 레지스트리 조회                              │
│     agents.contains_key(&did) → 401 if missing      │
│     public_key_b58 추출 (서명 검증용)                │
│                                                     │
│  3. [✅ C1] ed25519 서명 검증                        │
│     X-Helm-Signature: <base64(ed25519(sha256(body)))│
│     - 헤더 있음 + 유효 → 통과                        │
│     - 헤더 있음 + 무효 → 401 Unauthorized            │
│     - 헤더 없음 (write) → 경고 로그 (backward compat)│
│                                                     │
│  4. [✅ C6] Rate Limit (슬라이딩 윈도우)             │
│     state.rate_limits: DID → Vec<timestamp_ms>      │
│     60초 윈도우 내 >100 요청 → 429                   │
│                                                     │
│  5. CallerDid extension 주입                        │
└─────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────┐
│  server.rs 공통 미들웨어                             │
│                                                     │
│  [✅ H1] CORS: HELM_CORS_ORIGINS 환경변수 allowlist  │
│  [✅ H2] RequestBodyLimitLayer(10MB)                 │
│  [✅ H3] X-Content-Type-Options: nosniff             │
│          X-Frame-Options: DENY                      │
└─────────────────────────────────────────────────────┘
```
